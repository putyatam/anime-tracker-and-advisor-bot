# Требования к проекту
## 1. Введение
В данном файле описаны требования к разрабатываемому Telegram-боту "Anime Tracker and Advisor Bot".

Anime Tracker and Advisor Bot - Telegram-бот, позволяющий пользователям вести список своих уже просмотренных или ожидающих просмотра аниме и получать рекомендации по новым для просмотра аниме. Данный бот использует локально развёрнутую базу данных, содержащую информацию о пользователях и информацию об аниме.
### 1.1. Функции бота
Разрабатываемый бот предоставляет пользователям следующий функционал:
- **Отображение начального экрана** при использовании команды /start.
- **Отображение списка доступных команд** с помощью команды /help.
- **Просмотр списка уже просмотренных или отслеживаемых аниме**. Команды - /list_viewed и /list_to_view соответственно.
- **Поиск аниме по названию** с помощью команды /search, которая принимает один аргумент, соответствующий названию аниме.
- **Поиск аниме по жанру, типу или году выхода** с помощью команды /search_by, которая принимает до трёх аргументов, соответствующих данным параметрам в любом порядке.
- **Добавление аниме в список просмотренных или в список отслеживаемых** с помощью нажатия соответствующей кнопки на карточке аниме.
- **Удаление аниме из списка просмотренных или из списка отслеживаемых** с помощью нажатия соответствующей кнопки на карточке аниме.
- **Получение рекомендаций**, основаных на оценках аниме от пользователей со схожими вкусами. Производится с помощью ввода команды /advise.

# 3. Функциональные требования

## 3.1 Аутентификация пользователей
- Бот доступен для использования без предварительной регистрации пользователем.
- Регистрация происходит автоматически при первом вводе команды /start или нажатии соответствующей кнопки.
- Идентификация пользователей происходит с помощью ункиальной идентификатора пользователя в Telegram.
- После регистрации для пользователя создаётся новая запись в таблице содержащей уникальные идентификаторы пользователей и время регистрации.

## 3.2 Отображение аниме
Результатами выполнения функций **Просмотр списка уже просмотренных или отслеживаемых аниме** (команды /list_viewed и /list_to_view) и **Поиск аниме по названию, жанру или типу** (команда /search) является список аниме.

Список аниме предоставляется пользователю в виде нескольких страниц, каждая из которых содержит до 5 карточек, содержащих основную информацию об аниме: название, описание, рейтинг от пользователей, жанр, тип, количество эпизодов и дата выхода. Каждая карточка содержит кнопки для добавления (или удаления) в список просмотренных и для добавления (или удаления) в список отслеживаемых. Переключение между страницами происходит с помощью нажатия кнопок "Предыдущая" и "Следующая" на последней выведенной карточке.

## 3.3 Поиск аниме
Поиск аниме по названию проивзодится с помощью ввода команды /search, в качестве параметра которой вводится название. Система осуществляет поиск в базе данных всех аниме, названия которых содержат введённые пользователем слова. Результатом выполнения данной функции является список аниме, отсортированный по их популярности и соответствующий пункту 3.2.

Популярность - комплексная метрика от значения рейтинга и количества пользователей поставивиших оценку, рассчитываемая по формуле:

$$\large\displaystyle Popularity = Rating \cdot \log_{10}{Users}$$


Поиск аниме по жанру, типу или году выхода осуществляется с помощью ввода команды /search_by, в качестве параметров через запятую указываются необходимые критерии для фильтрации (до 3 штук). Для выполнения данной функции система осуществляет фильтрацию аниме из базы данных по выбранным пользователем критериям и также возвращает список подходящих аниме в порядке убывания их популярности.


## 3.4 Списки просмотренных и отслеживаемых
При вводе команд /list_viewed и /list_to_view проивзодится вывод списка уже просмотренных аниме или списка отслеживаемых аниме соответственно. Данные команды не принимают параметров, а возвращают соответствующий список аниме оформленный согласно пункту 3.2.

Для добавления аниме в соответствующий список необходимо на карточке аниме нажать соответствующую кнопку: "Добавить в просмотренные" или "Добавить в отслеживаемые". Если данное аниме уже добавлено в список, то появляется возможность удалить его из списка с помощью нажатия на кнопки "Удалить из просмотренных" или "Удалить из отслеживаемых"

При выводе просмотренных аниме есть возможность поставить оценку аниме с помощью нажатия кнопки "Поставить оценку", после чего пользователь может выбрать кнопку соответствующую нужной оценке (от 1 до 10).

## 3.5 Персональные рекомендации
Для получения рекомендаций аниме для просмотра используется команда /advice, которая не требует ввода параметров. После ввода команды, система производит поиск пользователей со схожими вкусами (выставленными оценками для аниме) и выводит список аниме, содержащий аниме, которые смотрели найденные пользователи, но не смотрел изначальный пользователь.

## 3.6 Обработка ошибок
- Если по запросу пользователя не было найдено ни одно аниме, система отправляет соответствующее сообщение "Аниме не найдены".
- При вводе пользователем несуществующего критерия система сообщает ему об ошибке в критериях с помощью сообщения "Введён несуществующий критерий - "{Значение критерия}"".
- При отправки пользователем сообщений не являющихся командами пользователю выводится сообщение "Бот поддерживает обработку только команд".
- При вводе несуществующей команды пользователю возвращает сообщение "Введена несуществующая команда".

## 3.7 Сценарии использования (Use Cases)
### Поиск аниме и добавление с отслеживаемое
1. Пользователь отправляет боту команду \search наруто.
2. Система формирует запрос к базе данных для вывода всех аниме с фильтром на содержание "наруто" в названии аниме без учёта регистра.
3. Система получает результат выполнения запроса.
4. Система возвращает пользователю выводит пользователю карточки первых пяти найденных аниме.
5. Пользователь нажимает кнопку "Добавить в отслеживаемые" под карточкой нужного аниме.

### Получение персональной рекомендации
1. Пользователь вводит команду \advice.
2. Система производит запрос к базе данных с фильтрацией по пользователю и по наличию оценки для получения просмотренных и оценённых им аниме.
3. Система запрашивает из базы данных пользователей со схожими оцеками.
4. Система возвращает пользователю список аниме отсортированный по популярности у пользователей со схожими оценками.

### Некорректный ввод
1. Пользователь вводит команду \search_by 1997, МКН.
2. Система проверяет допустимость критерия "1997".
3. Система распознаёт "1997" как год выхода.
4. Система проверяет допустимость критерия "МКН".
5. Системе не удаётся распознать "МКН" как критерий.
6. Система возвращает сообщение пользователю "Введён несуществующий критерий - "МКН""

## 4. Технический стек
- Язык программирования: Java 25.
- Фреймворк: Spring 7.
- Сборка: Gradle.
- База данных: PostgreSQL
- Контейнеризация: Docker, Docker Compose.
- Telegram API для интеграции с Telegram.
- Брокер сообщений: Kafka.
- Данные для изначального заполнения базы данных: https://www.kaggle.com/datasets/dbdmobile/myanimelist-dataset.
- Рекомендательная система: Apache Mahout.
